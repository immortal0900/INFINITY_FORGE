# {주제} 문제 {N}가지 — 쉽게 정리한 원인과 해결법

> 무엇에 관한 문서인가: {한 줄 요약 — 어떤 기능/영역의 어떤 상황에서 나는 문제인지}.
> 확인한 곳: `{조사한 코드 경로}` 를 직접 읽고 정리. 코드는 고치지 않았고, "어디가 문제이고 어떻게 고칠 수 있는지"만 담았습니다.
> 표의 코드 위치(파란 링크)를 클릭하면 실제 코드 줄로 이동합니다.

## 먼저 알아둘 기본 개념

이 몇 개만 알면 아래가 다 읽힙니다. (도메인 용어를 등장 즉시 풀이)

- **{용어1}**: {쉬운 풀이}.
- **{용어2}**: {쉬운 풀이}.

이 문서에서 **문제의 원인은 층위로 나뉩니다.** 이 구분이 핵심입니다.

- **코드(code)**: 프로그램이 실제로 실행하는 로직. → 고치면 **확정적**(무조건 그렇게 됨).
- **프롬프트(prompt)**: AI에게 사람이 글로 준 지시문. → 고치면 **확률적**(AI가 안 따를 수도).
- (해당되면) **설정/데이터**: config·스키마·매핑 테이블 등.

## 꼭 기억할 것
1. **원인이 코드냐 프롬프트냐를 구분하세요.** 코드 수정은 확정적, 프롬프트 수정은 확률적.
2. {SSoT 이중화 — 같은 로직이 여러 곳에 복사돼 있으면 함께 고쳐야 함, 있으면 기재}
3. {검증 사각지대 — 자동 테스트가 이 문제를 못 잡는 이유, 있으면 기재}
4. **버그인지 의도인지 코드만으론 확정 못 하는 항목**은 "먼저 확인할 것"으로 남겼습니다.
5. 여기 적은 해결법은 **위치와 방향 제안**입니다. 적용 전 회귀 테스트로 검증하세요.

# 기술 개념 설명 스타일

1. **전문용어 즉시 풀이**: 영문 원어 + 한국어 풀이 + 짧은 기능 설명을 괄호 안에 묶어 첫 등장 시 해결. 예: `PATH(전역 실행 경로, OS가 명령어 입력 시 뒤지는 폴더 목록)`
2. **나란히 비교표**: 두 개념 대조 시 표 사용, 여러 관점(무엇/어디에/크기/내용/용도)을 행으로 동시 비교
3. **구체적 수치·경로 제시**: 추상 설명에 머물지 말고 실제 파일 경로, 파일 크기, 명령 결과 같은 실측값 덧붙임
4. **비유 전면 금지**: 일상 사물·상황 비유(창고·리모컨·택배센터 등)로 기술 개념을 바꿔치지 마라. 비유가 떠오르면 대신 대상 자체의 동작·구조와 실제 코드·경로·수치로 서술하라. 개발지식이 없어도 논리적으로 이해되도록 짜임새 있게 직관적으로 서술할 것.
5. **단계 분해**: "1단계 → 2단계 → 3단계" 순서 명시, 무슨 일이 언제 일어나는지 분해
6. **직접 확인**: 사용자가 본인 환경에서 직접 확인해 볼 수 있게 한다
7. **왜 이렇게 하는지**: 설계 의도·대안 대비 장점·트레이드오프 포함
8. **한마디 요약**: 긴 설명 뒤에 한 문장 압축으로 닫기
9. **계층 구조**: `##` 소제목과 굵은 소개 문장으로 긴 설명을 탐색 가능하게 분할
10.**코드설명시**: 문법을 모르기 때문에 코드 설명 시, 모든 키워드(모든 코드 요소)에 '직역 주석'을 달 것
11.**직관성**: 최대한 직관적으로 설명.

## Why

기술 개념을 처음 접하는 관점에서도 막힘없이 읽히게 하는 게 목적.
